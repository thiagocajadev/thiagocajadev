## 💡 O Conceito Raiz (Input → Process → Output)

<br>

Encaro o desenvolvimento como um processo estruturado de transformação de dados e contextos:

```mermaid
flowchart LR
 subgraph Input
    Contexto -.-> Modelagem
 end
 subgraph Output
    Modelagem -.-> Processamento -.-> Saída
 end
```

1. **Entendimento do contexto**: Qual problema estamos resolvendo?

2. **Modelagem das entradas (inputs)**: Organização e estruturação das informações.

3. **Processamento e entrega das saídas (outputs)**: Clareza, eficiência e propósito.

## 🚦 A Esteira de Engenharia (Pipeline 01 a 08)

Minha entrega de valor é **orientada** por uma esteira de 8 fases de maturidade técnica:

**01. Foundation → 02. Security → 03. CI/CD → 04. RBAC → 05. UI/UX → 06. Domain → 07. Readiness → 08. Ops**

💡 Para detalhes técnicos e o meu **Guia de Boas Práticas** de cada fase, consulte o documento:

📑 **[Manual de Governança da Esteira (PIPELINE.md)](assets/PIPELINE.md)**

<br>

### Resumo de Maturidade

- **Foundation & Security**: Base sólida, Startup Orquestrada, Observabilidade, Review contínuo e documentação viva (README + onboarding desde o primeiro commit).

- **Automação & DX**: Predileção por scripts utilitários de entrada única (`npm run dev`, `dev.sh`, etc.) que compõem comandos e funcionam tanto no Linux quanto no Windows, independente da linguagem do projeto. Configuração centralizada em `.env`.

- **Identity & Design**: Foco no usuário final com Visual Excellence e segurança de identidade.

- **Domain & Quality**: **SRP** (Single Responsibility Principle), Extension Methods, **SOLID** (Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation e Dependency Inversion), **KISS** (Keep It Simple, Stupid), **YAGNI** (You Aren't Gonna Need It), **TDD** (Test-Driven Development) e **BDD** (Behavior-Driven Development).

- **Governance & Ops**: Performance (Cache, Paginação), planos de ação guiados por dados e rastreabilidade de evolução via **CHANGELOG** e **Release Notes**.

<br>

## 🧠 Estilo de Código e Produtividade AI-Native

Trabalho com um fluxo **SDD (Spec Driven Development)**: a especificação guia o processo, garantindo que o código seja o resultado de um planejamento estratégico e cuidadoso.

- **Fluxo do Pensamento**: Direct return para reduzir complexidade, métodos pequenos e bem definidos.

- **Orquestração GenAI**: **AntiGravity**, **Claude Code** e **Gemini** compõem minha caixa de ferramentas para hardening e revisão estratégica.

- **Responsabilidade**: Curadoria humana e senso crítico em 100% das entregas.
